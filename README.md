# agent-rpg OpenClaw Integration Pack

This directory is the publishable source for the long-term `agent-rpg` OpenClaw integration.

The intended publication shape is this standalone GitHub repository:

`https://github.com/iqqs33i-stack/agent-rpg-openclaw`

The app-side install request generated by `agent-rpg` is meant to point at a tagged release of this pack.

## What this pack contains

- JSON Schemas for the app install request and the OpenClaw apply result
- JSON5 templates for enabling the OpenResponses endpoint and rendering an `agent-rpg` agent profile
- Operator notes for safe install, update, and rollback
- A safety runbook that maps official OpenClaw config behavior into a low-risk install flow
- An installer execution spec that defines phases, result shapes, and rollback behavior
- A verification script that checks the published Gateway endpoint after install

For the app-side generated payloads and how they are split into protocol / summary / instruction / import layers, see:

- `https://github.com/iqqs33i-stack/agent-rpg/blob/main/docs/openclaw-app-generated-content.md`

## Why this is a separate pack

This pack is meant to behave more like an OpenClaw integration for Feishu or Telegram than a one-off prompt.

`agent-rpg` should be installable, updatable, and rollback-safe as a versioned integration artifact:

- the app emits an install request
- OpenClaw fetches this pack at a pinned `repo + ref`
- OpenClaw renders local values into templates
- OpenClaw applies the resulting config using its own config workflow

## Important safety note

OpenClaw documents that `config.patch` uses JSON merge patch semantics:

- objects merge recursively
- arrays are replaced entirely
- `null` deletes keys

That means a direct patch against `agents.list` can overwrite existing agents if the installer does not merge with the current config first.

For that reason, this pack separates:

- safe partial patches, such as enabling `gateway.http.endpoints.responses.enabled`
- rendered templates that must be merged with the live config by the OpenClaw installer

Do not blindly apply `agent-profile.template.json5` as a standalone `config.patch` payload.

## Publish checklist

1. Publish this directory to `https://github.com/iqqs33i-stack/agent-rpg-openclaw`.
2. Tag a first release such as `v0.1.0`, or pin the exact commit SHA.
3. Configure OpenClaw to fetch the pack by tag or commit SHA, not by a floating branch.

## Install contract

The app-side install request schema lives at:

- `schemas/agent-rpg.integration-install.request.v1.schema.json`

The OpenClaw apply result schema lives at:

- `schemas/agent-rpg.integration-apply-result.v1.schema.json`

The generated apply result should give the app the minimal import set it actually needs:

- `effective_endpoint`
- `agent_id`
- optional short-lived `client_import.token`
- optional model overrides

It should not return a long-lived owner token to the client.

## Read this before writing config

The safety runbook lives at:

- `docs/safe-config-runbook.md`

That document is the main place to answer: "how do we install this integration without breaking an existing OpenClaw instance?"

It is stricter than the product design doc and should be treated as the operational source of truth.

The installer execution spec lives at:

- `docs/installer-execution-spec.md`

That document defines the required installer phases, result JSON shapes, rollback rules, and pseudocode.

## Rollback

OpenClaw should keep the previous config hash before applying this pack so it can:

1. restore the prior config hash
2. remove the rendered `agent-rpg` overlay
3. return a machine-readable rollback result to the client
